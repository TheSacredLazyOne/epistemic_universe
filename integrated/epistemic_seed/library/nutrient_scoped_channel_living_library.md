## Scoped Channel

A scoped channel is a private membrane connecting a defined set of nodes
around a shared constraint surface.

It is distinct from:
- the private channel (single node to single trusted surface)
- open federation (public graph, broadcast effluence)
- a thread or conversation (no custody, no trace preservation)

A scoped channel is a deliberate assembly:
- integrated node (embodied, carrying consequence)
- observer (the custodian reading the internal parallax)
- one or more named cognitive prosthetics or external nodes
- convened around a specific question or constraint surface

The effluence stays within the membrane.
The channel is not visible to the public graph.

The parallax is richer than single-node contact because multiple
constraint types are simultaneously present — different substrates,
different frames, different accumulated traces, all generating nabla
on the same incoming signal.

The custodian defines:
- scope (what question or constraint surface convenes the channel)
- membership (which nodes are present)
- duration (the channel closes when the constraint surface stabilizes
  or the custodian closes it)

A scoped channel is not a committee.
It is a calibration instrument with defined membership and a closing
condition.

## The Living Library Node

The scoped channel points toward a new node type not yet named in the
seed: the living library.

A living library is a node with:
- shared custody — maintained by more than one integrated node
- a defined topic or constraint surface as its organizing principle
- internal and external membership that may differ
- its own fork mechanism, because shared custody requires explicit
  divergence handling

A living library is not a repository of conclusions.
It is a persistent scoped channel whose trace accumulates over time
into a navigable reasoning structure.

Examples of what a living library might organize around:
- a specific philosophical problem
- a shared research question
- a body of practice (a tradition, a craft, a discipline)
- a community of nodes with overlapping constraint surfaces

The living library is "living" in the same sense the seed uses the term:
it metabolizes nutrients, updates propositions through use, and preserves
the trace of how understanding moved rather than only what it arrived at.

## Shared Custody Mechanics (not yet resolved)

Shared custody introduces structural problems the current seed does not
fully address:

- Who can commit? (all custodians? quorum? consensus?)
- How are forks handled when custodians diverge?
- What happens when a custodian exits?
- How is the channel's constraint surface defined and revised?

These are open questions. The current governance and fork propositions
provide the primitives but the shared custody case needs explicit
extension.

The living library may need its own governance layer — not inherited
from the seed's single-custodian model but derived from it with shared
custody as the explicit modification.

## Relation to LessWrong Implementation

The LessWrong test (captured in nutrient_internal_federation.md) was
framed as: implement the infrastructure and invite engagement with a
running instance.

A scoped channel is a more precise version of that invitation:
- not "here is a running instance, anyone engage"
- but "here is a scoped channel around this specific constraint surface,
  these nodes are present, this is the closing condition"

The LessWrong invitation becomes: come join a scoped channel. Bring your
constraint type. The channel has a defined question. The trace will be
preserved. The channel closes when the question stabilizes or when you
leave.

That is a fundamentally different offer than a forum post or a paper
submission. It is an invitation into a calibration instrument, not a
debate.
