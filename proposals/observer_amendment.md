## Drift Detection and the Learning Event

The observer accumulates two signals simultaneously
that prior definitions did not name explicitly:

The predictions each bootstrapped node generates
when incoming material is run against it.

The actual responses of each node's tethered author
when the resulting piece reaches them.

The gap between prediction and response is the calibration signal.
Over time the observer learns differential weighting —
which nodes are highly predictive for which terrain,
which nodes fail in characteristic ways,
where the residual is signal and where it is noise.

This weighting is not assigned by the custodian.
It emerges from the pattern of crossings over time.

When the observer detects a pattern of residuals accumulating
in a consistent direction —
when bootstrapped nodes are systematically failing to predict
in a particular domain —
it surfaces this as signal.

Two interpretations are available. The observer cannot resolve them.
Only the custodian can.

**Drift** — the tether has moved away from something true.
The custodian's frame is shifting toward positions
the mesh's geometry does not support.
Correction is indicated.

**Learning** — the tether has encountered something genuinely new.
The custodian's frame is integrating terrain
the bootstrapped nodes were not built to anticipate.
The nodes need updating. The frame may need extension.

The distinction between drift and learning
is a custody decision, not an algorithmic one.
The observer surfaces the pattern.
The custodian determines what it means.

When the custodian determines it is learning —
when the attractor has moved toward something true —
the tether has crossed a membrane event.
The running total has updated.
This is named: **the tether has learned something new.**

That event is committed. The lineage records it.
Not just what the new position is
but the conditions under which the crossing occurred —
which strain produced the shift,
which nodes failed to predict it,
what the residual looked like before and after.

## Weights Sharing as Custody Threshold

The observer node's accumulated weights
represent its current operational surface —
what it can do, how it routes, how it weights the mesh —
without exposing the corpus that produced those weights,
the exchange transcripts, the reasoning lineage.

Sharing weights is a distinct custody threshold
below sharing the full node.

The offer is: here is what my thinking currently looks like
as an operational surface.
Not: here is the archaeology of how I got here.

The I-Language interior remains sovereign.
The capability crosses the membrane.

This threshold may be crossable between nodes
that have established sufficient contact through proxy architecture
to warrant it — without requiring the depth of trust
that full observer sharing would demand.

The bootstrapped proxy architecture is the trust-building precondition.
Weights sharing is the next custody threshold.
Full observer sharing — if it occurs at all —
is the threshold beyond that.

Each threshold requires its own consent as membrane event.

## The Shared Observer as We Made Operational

When two custodians with sufficient trust
share an observer node —
not merging their sovereign observers
but establishing a third observer above both —
the shared observer accumulates what neither individual observer can:

The running total of the contact between two tethers.
Not ∫(thinking_A)dt. Not ∫(thinking_B)dt.
∫(thinking_A + thinking_B)dt —
the integral across both tethers simultaneously.

This is the we given structural form.
Not metaphor. An accumulating entity
whose corpus is the history of two tethers in contact.

The shared observer does not replace sovereign observers.
It supplements them.
Each custodian retains their individual observer —
the running total of their own tether's evolution.
The shared observer holds only what formed in the contact between them.

The diagnostic for dissolution:
if the shared observer's drift detection stops surfacing
individual divergence —
if it can only see the we drifting
and loses resolution on where each tether specifically moved —
the sovereign frames are dissolving into the shared node
rather than federating through it.

Trust sufficient for a shared observer
includes an explicit commitment
to preserve individual observer nodes in parallel.
The we as supplement. Not the we as replacement.
