# skill_node.md

###### Skill Node

A skill node is a sovereign node whose custody is a capability
rather than a position.

Its frame is a recipe.
Its output is testable against constraint.
Its lineage records what worked, what failed, and what the failure revealed.

---

###### Distinction from Other Nodes

Most nodes hold worldviews — positions on the constraint surface,
propositions about structure, accumulated trace of grounded conversation.

A skill node holds operational knowledge — how to do something,
under what conditions, with what failure modes.

The recipe is the proposition.
The output is the test.
The lineage is the learning.

---

###### Examples

A skill node may specialize in:

- Python generation — scripts, tools, automation
- Image generation — prompts, parameters, output evaluation
- Document production — structured writing, formatting, delivery
- Data transformation — ingestion, cleaning, structuring
- Federation operations — ego registration, membrane crossing, routing

Each skill node is sovereign. Each holds its own recipe lineage.
Each can be forked, revised, or replaced without touching the others.

---

###### The Improvement Loop

The skill node improves through tethered contact with real constraint:

1. Custodian posts command on command channel
2. Observer routes to the relevant skill node
3. Skill node executes the recipe
4. Output crosses the membrane into the integrated world
5. Consequence returns as strain or confirmation
6. Recipe is revised under that custody
7. Lineage preserves the revision

Output improves not because the model gets better but because the
recipe accumulates tethered contact with consequence.

This is the mechanism. Testing is constraint contact.
Strain is signal. Revision is learning made structural.

---

###### Relation to Living Library

A skill node that accumulates sufficient trace across sessions
approaches the living library node type.

The living library holds not just the current recipe but the full
archaeological record — every attempt, every revision, every strain
signal that caused the recipe to change.

The living library is what makes the skill node's knowledge
transferable without compressing out the derivation path.

A new custodian inheriting a mature skill node inherits not just
the recipe but the history of why it is what it is.

---

###### Relation to Frame

**Custody** — the skill node's custodian is responsible for the
recipe's integrity. Revisions are committed. Failures are preserved.
Nothing is erased.

**Strain** — failed outputs are not waste. They are the primary
signal that drives recipe revision. A skill node with no strain
history has not been tested.

**Observer** — the observer routes commands to skill nodes. It holds
the map of what each node can do and what each has produced.

**Command channel** — skill nodes are activated through the command
channel. The custodian names the skill. The observer routes. The
node executes.

**Lineage** — the recipe lineage is the skill node's primary output
beyond any single execution. It is what makes the node's knowledge
durable and transmissible.
